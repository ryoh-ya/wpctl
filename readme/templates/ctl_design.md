# CLIツール設計仕様書

## CLIツールの概要

WordpressのAPIを活用して、記事を投稿、および管理できるツール

- **ツール名**: wpctl

## ツール機能

| 分類     | 機能名                | 説明                       | 必要 | 現状 |
| -------- | --------------------- | -------------------------- | ---- | ---- |
| 記事管理 | Wordpress記事投稿機能 | 記事を投稿する機能         | MUST |      |
|          | Wordpress記事更新機能 | 記事を更新する機能         | MUST |      |
| 共通機能 | MarkDown>HTTP変換機能 | MD形式をHTML形式に変換する | MUST |      |

## インターフェース

- **コマンド名**: `wpctl`
- **サブコマンド**:
  - `post`: 記事を管理する機能
    - `create`: 記事を投稿する
    - `update`: 記事を更新数る


### `post　create`

記事を投稿する

```sh
wpctl post create [-t "記事のタイトル" ] <FilePath> 
```

**入力値**

| 入力値     | 引数           | 必須 | 内容                                                   |
| ---------- | -------------- | ---- | ------------------------------------------------------ |
| `FilePath` |                | true | 記事投稿するファイルパスを設定する                     |
| `title`    | `--title`,`-t` |      | 記事のタイトルを指定する(デフォルト`タイトル未設定`) |
|            |                |      |                                                        |

**出力値**

`Wordpress API`のレスポンスデータ


### `post　update`

記事を投稿する

```sh
wpctl post  update -id 1.  [-t "記事のタイトル"] <FilePath>
```

**入力値**

| 入力値     | 引数           | 必須 | 内容                                                 |
| ---------- | -------------- | ---- | ---------------------------------------------------- |
| `FilePath` |                | true | 記事投稿するファイルパスを設定する                   |
| `id`       | `--d`          | true | 更新する場合にidを指定します                         |
| `title`    | `--title`,`-t` |      | 記事のタイトルを指定する(デフォルト`タイトル未設定`) |

**出力値**

`Wordpress API`のレスポンスデータ


## 環境変数

| 環境変数        | 名前             | 必須 | 内容                                          |
| --------------- | ---------------- | ---- | --------------------------------------------- |
| WP_USER         | ユーザー名       | true | Wordpressに投稿するユーザー名を指定する       |
| WP_APP_PASSWORD | アプリパスワード | true | Wordpressで設定するアプリパスワードを指定する |


## 基本仕様

### 入力に関する仕様

- 対応ファイルは`.txt`,`.md`,`.html`
- 環境変数は`.env`に対応しないものとする(あらかじめ環境変数に設定しておく必要がある)
