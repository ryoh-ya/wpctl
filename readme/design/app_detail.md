# 詳細設計

## 共通事項

- １つのファイルが肥大化しないように役割に応じ分割すること
- ドキュコメントはGoogle方式(Python Docstring)により記載すること。

## エンドポイント周りの詳細設計

**仕様**
- ツールは`main.py`から実行されること
- `main.py`には主にコマンドライン引数、ログの初期化などを処理する
- `main.py`から`run.py`の`run`メソッドに引数を渡すようにする
- `run.py`の`run`メソッドよりツールのロジックが開始されるようにする


## ディレクトリ設計

- `api`: 外部連携を行うAPI集
  - 基本的には外部APIの仕様通りに実装する
- `commands`: CLIのコマンドについて実装する
  - コマンドの実際のロジックを実装する
- `utils`: ユーティリティ・ヘルパー関数などを実装する(共通化できる一般的なコード集)
  - アプリ・ツールの仕様が変わっても変更がない共通処理などを実装する
- `libs`: 全体の共通処理を実装する(アプリ・ツールの仕様が変わったら変更が必要な部品)

## 機能設計

### ファイル読み込み処理・MarkDown>HTTP変換機能

**仕様**

- `.txt`、`.html`の場合はそのままAPIに渡す。
- `markdown`の場合はHTMLに変換してAPIに渡す
  - `<script>` タグは許可しない
  - `onerror`, `onclick` などのイベント属性は許可しない
  - 外部JavaScriptの埋め込みは許可しない
- `libs`に実装することとする

### Wordpress記事投稿機能(Create)

**仕様**
- Wordpress APIを呼び出して投稿する
- エラーの場合に必ずユーザーがログでわかること
- 現時点ではカテゴリは考慮しない

### Wordpress記事更新機能(Update)


**仕様**
- Wordpress APIを呼び出して更新する
- エラーの場合に必ずユーザーがログでわかること
- 現時点ではカテゴリは考慮しない
